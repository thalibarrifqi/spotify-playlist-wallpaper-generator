# Engineering Principles

## General

- Keep every sprint small.
- Prefer native browser APIs over extra libraries.
- Install dependencies only when needed.
- Review AI proposals before approving changes.
- Every sprint must have clear acceptance criteria.

## AI Workflow

1. Define the Sprint Goal.
2. Write the requirements.
3. Define what is out of scope.
4. Ask OpenCode for an implementation plan.
5. Review the proposal.
6. Approve implementation.
7. Request a code review.
8. Reflect before starting the next sprint.
